087/**
088 * A distributed {@link org.jsimpledb.kv.KVDatabase} based on the Raft consensus algorithm.
089 *
090 * <p>
091 * Raft defines a distributed consensus algorithm for maintaining a shared state machine.
092 * Each Raft node maintains a complete copy of the state machine. Cluster nodes elect a
093 * leader who collects and distributes updates and provides for consistent reads.
094 * As long as as a node is part of a majority, the state machine is fully operational.
095 *
096 * <p>
097 * {@link RaftKVDatabase} turns this into a transactional key/value database with linearizable ACID semantics as follows:
098 *  <ul>
099 *  <li>The Raft state machine is the key/value store data.</li>
100 *  <li>Unapplied log entries are stored on disk as serialized mutations, and also cached in memory.</li>
101 *  <li>On leaders only, committed log entries are not applied immediately; instead they kept around for time at least
102 *      <i>T<sub>max</sub></i> since creation, where <i>T<sub>max</sub></i> is the maximum supported transaction duration.
103 *      This caching is subject to <i>M<sub>max</sub></i>, which limits the total memory used; if reached, these log entries
104 *      are applied early.</li>
105 *  <li>Concurrent transactions are supported through a simple optimistic locking MVCC scheme (same as used by
106 *      {@link org.jsimpledb.kv.mvcc.SnapshotKVDatabase}):
107 *      <ul>
108 *      <li>Transactions execute locally until commit time, using a {@link MutableView} to collect mutations.
109 *          The {@link MutableView} is based on the local node's most recent log entry (whether committed or not);
110 *          this is called the <i>base term and index</i> for the transaction.</li>
111 *      <li>On commit, the transaction's {@link Reads}, {@link Writes}, base index and term, and any config change are
112 *          {@linkplain CommitRequest sent} to the leader.</li>
113 *      <li>The leader confirms that the log entry corresponding to the transaction's base index is either not yet applied,
114 *          or was its most recently applied log entry. If this is not the case, then the transaction's base log entry
115 *          is too old (older than <i>T<sub>max</sub></i>, or was applied and discarded early due to memory pressure),
116 *          and so the transaction is rejected with a {@link RetryTransactionException}.
117 *      <li>The leader verifies that the the log entry term matches the transaction's base term; if not, the base log entry
118 *          has been overwritten, and the transaction is rejected with a {@link RetryTransactionException}.
119 *      <li>The leader confirms that the {@link Writes} associated with log entries after the transaction's base log entry
120 *          do not create {@linkplain Reads#isConflict conflicts} when compared against the transaction's {@link Reads}.
121 *          If so, the transaction is rejected with a {@link RetryTransactionException}.</li>
122 *      <li>The leader adds a new log entry consisting of the transaction's {@link Writes} (and any config change) to its log.
123 *          The associated term and index become the transaction's <i>commit term and index</i>; the leader then
124 *          {@linkplain CommitResponse replies} to the follower with this information.</li>
125 *      <li>If/when the follower sees a <i>committed</i> (in the Raft sense) log entry appear in its log matching the
126 *          transaction's commit term and index, then the transaction is complete.</li>
127 *      <li>As an optimization, when the leader sends a log entry to the same follower who committed the corresponding
128 *          transaction in the first place, only the transaction ID is sent, because the follower already has the data.</li>
129 *      </ul>
130 *  </li>
131 *  <li>For transactions occurring on a leader, the logic is similar except of course no network communication occurs.</li>
132 *  <li>For read-only transactions, the leader does not create a new log entry; instead, the transaction's commit
133 *      term and index are set to the base term and index, and the leader also calculates its current "leader lease timeout",
134 *      which is the earliest time at which it is possible for another leader to be elected.
135 *      This is calculated as the time in the past at which the leader sent {@link AppendRequest}'s to a majority of followers
136 *      who have since responded, plus the {@linkplain #setMinElectionTimeout minimum election timeout}, minus a small adjustment
137 *      for possible clock drift (this assumes all nodes have the same minimum election timeout configured). If the current
138 *      time is prior to the leader lease timeout, the transaction may be committed as soon as log entry corresponding to the
139 *      commit term and index is committed (it may already be); otherwise, the current time is returned to the follower
140 *      as minimum required leader lease timeout before the transaction may be committed.</li>
141 *  <li>Every {@link AppendRequest} includes the leader's current timestamp and leader lease timeout, so followers can commit
142 *      any waiting read-only transactions. Leaders keep track of which followers are waiting on which leader lease
143 *      timeout values, and when the leader lease timeout advances to allow a follower to commit a transaction, the follower
144 *      is immediately notified.</li>
145 *  <li>A weaker consistency guarantee for read-only transactions (stale reads) is also possible; see
146 *      {@link RaftKVTransaction#setStaleReadOnly RaftKVTransaction.setStaleReadOnly()}; these transactions generate
147 *      no additional network traffic.</li>
148 *  </ul>
149 *
150 * <p><b>Limitations</b></p>
151 *
152 * <ul>
153 *  <li>A transaction's mutations must fit in memory.</li>
154 *  <li>An {@link AtomicKVStore} is required to store local persistent state (note: any {@link KVDatabase} can be converted
155 *      into an {@link AtomicKVStore} using the {@link org.jsimpledb.kv.mvcc.AtomicKVDatabase} utility class)</li>
156 *  <li>All nodes must be configured with the same {@linkplain #setMinElectionTimeout minimum election timeout}.
157 *      This guarantees that the leader's lease timeout calculation is valid.</li>
158 *  <li>Due to the optimistic locking approach used, this implementation will perform poorly when there is a high
159 *      rate of conflicting transactions; the result will be many transaction retries.</li>
160 *  <li>Performance will suffer when the amount of data associated with a typical transaction cannot be delivered
161 *      quickly and reliably over the network.</li>
162 * </ul>
163 *
164 * <p>
165 * In general, the algorithm should function correctly under all non-Byzantine conditions. The level of difficultly
166 * the system is experiencing, due to contention, network errors, etc., can be measured in terms of:
167 * <ul>
168 *  <li>The average amount of time it takes to commit a transaction</li>
169 *  <li>The frequency of {@link RetryTransactionException}'s</li>
170 * </ul>
171 *
172 * <p><b>Cluster Configuration</b></p>
173 *
174 * <p>
175 * Instances support dynamic cluster configuration changes at runtime.
176 *
177 * <p>
178 * Initially, all nodes are in an <i>unconfigured</i> state, where nothing has been added to the Raft log yet and no
179 * cluster is defined. Unconfigured nodes are passive: they stay in follower mode (i.e., they will not start elections),
180 * and they disallow local transactions that make any changes other than as described below to initialize a new cluster.
181 *
182 * <p>
183 * An unconfigured node becomes configured when either:
184 * <ol>
185 *  <li>{@link RaftKVTransaction#configChange RaftKVTransaction.configChange()} is invoked and committed within
186 *      a local transaction, which creates a new single node cluster and commits the first log entry; or</li>
187 *  <li>An {@link AppendRequest} is received from a leader of some existing cluster, in which case the node
188 *      records the cluster ID (see below) and applies the received cluster configuration</li>
189 * </ol>
190 *
191 * <p>
192 * A node is configured if and only if it has recorded one or more log entries. The very first log entry
193 * always contains the initial cluster configuration (containing only the node that created it), so any node that has a
194 * non-empty log is configured.
195 *
196 * <p>
197 * Newly created clusters are assigned a random 32-bit cluster ID (option #1 above). This ID is included in all messages sent
198 * over the network, and adopted by unconfigured nodes that join the cluster (via option #2 above). Nodes discard incoming
199 * messages containing a cluster ID different from one they have seen previously. This prevents data corruption that can occur
200 * if nodes from two different clusters are inadvertently "mixed" together.
201 *
202 * <p>
203 * Once a node joins a cluster with a specific cluster ID, it cannot be reassigned to a different cluster without first
204 * returning it to the unconfigured state; to do that, it must be shut it down and its persistent state deleted.
205 *
206 * <p><b>Configuration Changes</b></p>
207 *
208 * <p>
209 * Once a node configured, a separate issue is whether the node is <i>included</i> in its own configuration, i.e., whether
210 * the node is a member of its cluster. A node that is not a member of its cluster does not count its own vote to determine
211 * committed log entries (if a leader), and does not start elections (if a follower). However, it will accept and respond
212 * to incoming {@link AppendRequest}s and {@link RequestVote}s.
213 *
214 * <p>
215 * In addition, leaders follow these rules:
216 * <ul>
217 *  <li>If a leader is removed from a cluster, it remains the leader until the configuration change that
218 *      removed it is committed (not counting its own vote), and then steps down (reverts to follower).</li>
219 *  <li>If a follower is added to a cluster, the leader immediately starts sending that follower {@link AppendRequest}s.</li>
220 *  <li>If a follower is removed from a cluster, the leader continues to send that follower {@link AppendRequest}s
221 *      until the follower acknowledges receipt of the log entry containing the configuration change.</li>
222 *  <li>Configuration changes that remove the last node in a cluster are disallowed.</li>
223 *  <li>Only one configuration change may take place at a time.</li>
224 * </ul>
225 *
226 * @see <a href="https://raftconsensus.github.io/">The Raft Consensus Algorithm</a>
227 */
228public class RaftKVDatabase implements KVDatabase {

748    @Override
749    @PostConstruct
750    public synchronized void start() {
751
752        // Sanity check
753        assert this.checkState();
754        if (this.role != null)
755            return;
756        Preconditions.checkState(!this.shuttingDown, "shutdown in progress");
757        Preconditions.checkState(this.logDir != null, "no log directory configured");
758        Preconditions.checkState(this.kv != null, "no local persistence key/value store configured");
759        Preconditions.checkState(this.network != null, "no network configured");
760        Preconditions.checkState(this.minElectionTimeout <= this.maxElectionTimeout, "minElectionTimeout > maxElectionTimeout");
761        Preconditions.checkState(this.heartbeatTimeout < this.minElectionTimeout, "heartbeatTimeout >= minElectionTimeout");
762        Preconditions.checkState(this.identity != null, "no identity configured");
763
764        // Log
765        if (this.log.isDebugEnabled())
766            this.debug("starting " + this.getClass().getName() + " in directory " + this.logDir);
767
768        // Start up local database
769        boolean success = false;
770        try {
771
772            // Create/verify log directory
773            if (!this.logDir.exists() && !this.logDir.isDirectory())
774                throw new IOException("failed to create directory `" + this.logDir + "'");
775            if (!this.logDir.isDirectory())
776                throw new IOException("file `" + this.logDir + "' is not a directory");
777
778            // Start k/v store
779            this.kv.start();
780
781            // Open directory containing log entry files so we have a way to fsync() it
782            assert this.logDirChannel == null;
783            this.logDirChannel = FileChannel.open(this.logDir.toPath(), StandardOpenOption.READ);
784
785            // Create randomizer
786            assert this.random == null;
787            this.random = new SecureRandom();
788
789            // Start up executor thread
790            assert this.executor == null;
791            this.executor = Executors.newSingleThreadScheduledExecutor(new ThreadFactory() {
792                @Override
793                public Thread newThread(Runnable action) {
794                    return new ServiceThread(action);
795                }
796            });
797
798            // Start network
799            this.network.start(new Network.Handler() {
800                @Override
801                public void handle(String sender, ByteBuffer buf) {
802                    RaftKVDatabase.this.handle(sender, buf);
803                }
804
805                @Override
806                public void outputQueueEmpty(String address) {
807                    RaftKVDatabase.this.outputQueueEmpty(address);
808                }
809            });
810
811            // Reload persistent raft info
812            this.clusterId = (int)this.decodeLong(CLUSTER_ID_KEY, 0);
813            this.currentTerm = this.decodeLong(CURRENT_TERM_KEY, 0);
814            final String votedFor = this.decodeString(VOTED_FOR_KEY, null);
815            this.lastAppliedTerm = this.decodeLong(LAST_APPLIED_TERM_KEY, 0);
816            this.lastAppliedIndex = this.decodeLong(LAST_APPLIED_INDEX_KEY, 0);
817            this.lastAppliedConfig = this.decodeConfig(LAST_APPLIED_CONFIG_KEY);
818            this.currentConfig = this.buildCurrentConfig();
819
820            // If we crashed part way through a snapshot install, recover by resetting state machine
821            if (this.lastAppliedTerm < 0 || this.lastAppliedIndex < 0) {
822                if (this.log.isDebugEnabled())
823                    this.debug("detected partially applied snapshot, resetting state machine");
824                if (!this.resetStateMachine(true))
825                    throw new IOException("error resetting state machine");
826            }
827
828            // Initialize commit index
829            this.commitIndex = this.lastAppliedIndex;
830
831            // Reload outstanding log entries from disk
832            this.loadLog();
833
834            // Show recovered state
835            if (this.log.isDebugEnabled()) {
836                this.debug("recovered Raft state:"
837                  + "\n  clusterId=" + (this.clusterId != 0 ? String.format("0x%08x", this.clusterId) : "none")
838                  + "\n  currentTerm=" + this.currentTerm
839                  + "\n  lastApplied=" + this.lastAppliedIndex + "t" + this.lastAppliedTerm
840                  + "\n  lastAppliedConfig=" + this.lastAppliedConfig
841                  + "\n  currentConfig=" + this.currentConfig
842                  + "\n  votedFor=" + (votedFor != null ? "\"" + votedFor + "\"" : "nobody")
843                  + "\n  log=" + this.raftLog);
844            }
845
846            // Validate recovered state
847            if (this.isConfigured()) {
848                Preconditions.checkArgument(this.clusterId != 0);
849                Preconditions.checkArgument(this.currentTerm > 0);
850                Preconditions.checkArgument(this.getLastLogTerm() > 0);
851                Preconditions.checkArgument(this.getLastLogIndex() > 0);
852                Preconditions.checkArgument(!this.currentConfig.isEmpty());
853            } else {
854                Preconditions.checkArgument(this.lastAppliedTerm == 0);
855                Preconditions.checkArgument(this.lastAppliedIndex == 0);
856                Preconditions.checkArgument(this.getLastLogTerm() == 0);
857                Preconditions.checkArgument(this.getLastLogIndex() == 0);
858                Preconditions.checkArgument(this.currentConfig.isEmpty());
859                Preconditions.checkArgument(this.raftLog.isEmpty());
860            }
861
862            // Start as follower (with unknown leader)
863            this.changeRole(new FollowerRole(this, null, null, votedFor));
864
865            // Done
866            this.info("successfully started " + this + " in directory " + this.logDir);
867            success = true;
868        } catch (IOException e) {
869            throw new RuntimeException("error starting up database", e);
870        } finally {
871            if (!success)
872                this.cleanup();
873        }
874
875        // Sanity check
876        assert this.checkState();
877    }

879    @Override
880    @PreDestroy
881    public void stop() {
882
883        // Set flag to prevent new transactions
884        synchronized (this) {
885
886            // Sanity check
887            assert this.checkState();
888            if (this.role == null || this.shuttingDown)
889                return;
890
891            // Set shutting down flag
892            this.info("starting shutdown of " + this);
893            this.shuttingDown = true;
894
895            // Fail all remaining open transactions
896            for (RaftKVTransaction tx : new ArrayList<RaftKVTransaction>(this.openTransactions.values())) {
897                switch (tx.getState()) {
898                case EXECUTING:
899                    tx.rollback();
900                    break;
901                case COMMIT_READY:
902                case COMMIT_WAITING:
903                    this.fail(tx, new KVTransactionException(tx, "database shutdown"));
904                    break;
905                case COMPLETED:
906                    break;
907                default:
908                    assert false;
909                    break;
910                }
911            }
912
913            // Sleep while we wait for transactions to clean themselves up
914            boolean done = false;
915            try {
916                done = TimedWait.wait(this, 5000, new org.dellroad.stuff.java.Predicate() {
917                    @Override
918                    public boolean test() {
919                        return RaftKVDatabase.this.openTransactions.isEmpty();
920                    }
921                });
922            } catch (InterruptedException e) {
923                Thread.currentThread().interrupt();
924            }
925            if (!done)
926                this.warn("open transactions not cleaned up during shutdown");
927        }
928
929        // Shut down the executor and wait for pending tasks to finish
930        this.executor.shutdownNow();
931        try {
932            this.executor.awaitTermination(1000, TimeUnit.MILLISECONDS);
933        } catch (InterruptedException e) {
934            Thread.currentThread().interrupt();
935        }
936
937        // Final cleanup
938        synchronized (this) {
939            this.executor = null;
940            this.cleanup();
941        }
942
943        // Done
944        this.info("completed shutdown of " + this);
945    }

948        assert Thread.holdsLock(this);
949        assert this.openTransactions.isEmpty();
950        if (this.role != null) {
951            this.role.shutdown();
952            this.role = null;

981    /**
982     * Initialize our in-memory state from the persistent state reloaded from disk.
983     * This is invoked on initial startup.
984     */
985    private void loadLog() throws IOException {
986
987        // Sanity check
988        assert Thread.holdsLock(this);
989        assert this.raftLog.isEmpty();
990
991        // Scan for log entry files
992        this.raftLog.clear();
993        for (File file : this.logDir.listFiles()) {
994
995            // Ignore sub-directories (typically owned by the underlying k/v store)
996            if (file.isDirectory())
997                continue;
998
999            // Is this a log entry file?
1000            if (LogEntry.LOG_FILE_PATTERN.matcher(file.getName()).matches()) {
1001                if (this.log.isDebugEnabled())
1002                    this.debug("recovering log file " + file.getName());
1003                final LogEntry logEntry = LogEntry.fromFile(file);
1004                this.raftLog.add(logEntry);
1005                continue;
1006            }
1007
1008            // Is this a leftover temporary file?
1009            if (TEMP_FILE_PATTERN.matcher(file.getName()).matches()) {
1010                if (this.log.isDebugEnabled())
1011                    this.debug("deleting leftover temporary file " + file.getName());
1012                if (!file.delete())
1013                    this.error("failed to delete leftover temporary file " + file);
1014                continue;
1015            }
1016
1017            // Unknown
1018            this.warn("ignoring unrecognized file " + file.getName() + " in my log directory");
1019        }
1020
1021        // Verify we have a contiguous range of log entries starting from the snapshot index; discard bogus log files
1022        Collections.sort(this.raftLog, LogEntry.SORT_BY_INDEX);
1023        long lastTermSeen = this.lastAppliedTerm;
1024        long expectedIndex = this.lastAppliedIndex + 1;
1025        for (Iterator<LogEntry> i = this.raftLog.iterator(); i.hasNext(); ) {
1026            final LogEntry logEntry = i.next();
1027            String error = null;
1028            if (logEntry.getTerm() < lastTermSeen)
1029                error = "term " + logEntry.getTerm() + " < last applied term " + lastTermSeen;
1030            else if (logEntry.getIndex() < this.lastAppliedIndex)
1031                error = "index " + logEntry.getIndex() + " < last applied index " + this.lastAppliedIndex;
1032            else if (logEntry.getIndex() != expectedIndex)
1033                error = "index " + logEntry.getIndex() + " != expected index " + expectedIndex;
1034            if (error != null) {
1035                this.warn("deleting bogus log file " + logEntry.getFile().getName() + ": " + error);
1036                if (!logEntry.getFile().delete())
1037                    this.error("failed to delete bogus log file " + logEntry.getFile());
1038                i.remove();
1039            } else {
1040                expectedIndex++;
1041                lastTermSeen = logEntry.getTerm();
1042            }
1043        }
1044        if (this.log.isDebugEnabled()) {
1045            this.debug("recovered " + this.raftLog.size() + " log entries: " + this.raftLog
1046              + " (" + this.getUnappliedLogMemoryUsage() + " total bytes)");
1047        }
1048
1049        // Rebuild current configuration
1050        this.currentConfig = this.buildCurrentConfig();
1051    }

1054
1055        // Start with last applied config
1056        final HashMap<String, String> config = new HashMap<>(this.lastAppliedConfig);
1057

1100    /**
1101     * Commit a transaction.
1102     */
1103    void commit(final RaftKVTransaction tx) {
1104        try {
1105
1106            // Mark transaction as "commit ready" - service thread will do the rest
1107            synchronized (this) {
1108
1109                // Sanity check
1110                assert this.checkState();
1111                assert this.role != null;
1112
1113                // Check tx state
1114                switch (tx.getState()) {
1115                case EXECUTING:
1116
1117                    // Transition to COMMIT_READY state
1118                    if (this.log.isDebugEnabled())
1119                        this.debug("committing transaction " + tx);
1120                    tx.setState(TxState.COMMIT_READY);
1121                    this.requestService(this.role.new CheckReadyTransactionService(tx));
1122
1123                    // Setup commit timer
1124                    final int timeout = tx.getTimeout();
1125                    if (timeout != 0) {
1126                        final Timer commitTimer = new Timer("commit timer for " + tx,
1127                          new Service(null, "commit timeout for tx#" + tx.getTxId()) {
1128                            @Override
1129                            public void run() {
1130                                switch (tx.getState()) {
1131                                case COMMIT_READY:
1132                                case COMMIT_WAITING:
1133                                    RaftKVDatabase.this.fail(tx, new RetryTransactionException(tx,
1134                                      "transaction failed to complete within " + timeout + "ms (in state " + tx.getState() + ")"));
1135                                    break;
1136                                default:
1137                                    break;
1138                                }
1139                            }
1140                        });
1141                        commitTimer.timeoutAfter(tx.getTimeout());
1142                        tx.setCommitTimer(commitTimer);
1143                    }
1144                    break;
1145                case CLOSED:                                        // this transaction has already been committed or rolled back
1146                    throw new StaleTransactionException(tx);
1147                default:                                            // another thread is already doing the commit
1148                    this.warn("simultaneous commit()'s requested for " + tx + " by two different threads");
1149                    break;
1150                }
1151            }
1152
1153            // Wait for completion
1154            try {
1155                tx.getCommitFuture().get();
1156            } catch (InterruptedException e) {
1157                throw new RetryTransactionException(tx, "thread interrupted while waiting for commit", e);
1158            } catch (ExecutionException e) {
1159                final Throwable cause = e.getCause();
1160                Util.prependCurrentStackTrace(cause, "Asynchronous Commit");
1161                if (cause instanceof Error)
1162                    throw (Error)cause;
1163                if (cause instanceof RuntimeException)
1164                    throw (RuntimeException)cause;
1165                throw new KVTransactionException(tx, "commit failed", cause);           // should never get here
1166            }
1167        } finally {
1168            this.cleanupTransaction(tx);
1169        }
1170    }

1173     * Rollback a transaction.
1174     */
1175    synchronized void rollback(RaftKVTransaction tx) {
1176
1177        // Sanity check
1178        assert this.checkState();
1179        assert this.role != null;
1180
1181        // Check tx state
1182        switch (tx.getState()) {
1183        case EXECUTING:
1184            if (this.log.isDebugEnabled())
1185                this.debug("rolling back transaction " + tx);
1186            this.cleanupTransaction(tx);
1187            break;
1188        case CLOSED:
1189            break;
1190        default:                                            // another thread is currently committing!
1191            this.warn("simultaneous commit() and rollback() requested for " + tx + " by two different threads");
1192            break;
1193        }
1194    }

1196    private synchronized void cleanupTransaction(RaftKVTransaction tx) {
1197
1198        // Debug
1199        if (this.log.isTraceEnabled())
1200            this.trace("cleaning up transaction " + tx);
1201
1202        // Do any per-role cleanups
1203        if (this.role != null)
1204            this.role.cleanupForTransaction(tx);
1205
1206        // Cancel commit timer
1207        final Timer commitTimer = tx.getCommitTimer();
1208        if (commitTimer != null)
1209            commitTimer.cancel();
1210
1211        // Remove from open transactions set
1212        this.openTransactions.remove(tx.getTxId());
1213
1214        // Transition to CLOSED
1215        tx.setState(TxState.CLOSED);
1216
1217        // Notify waiting thread if doing shutdown
1218        if (this.shuttingDown)
1219            this.notify();
1220    }

1224

1259     * A view of the database based on the most recent log entry, if any, otherwise directly on the committed key/value store.
1260     * Caller is responsible for eventually closing the snapshot.
1261     */
1262    private class MostRecentView {
1263
1264        private final CloseableKVStore snapshot;
1265        private final long term;
1266        private final long index;
1267        private final HashMap<String, String> config;
1268        private final MutableView view;
1269
1270        public MostRecentView() {
1271            assert Thread.holdsLock(RaftKVDatabase.this);
1272
1273            // Grab a snapshot of the key/value store
1274            this.snapshot = RaftKVDatabase.this.kv.snapshot();

1282                    kview = new MutableView(kview, null, logEntry.getWrites());
1283                logEntry.applyConfigChange(this.config);
1284            }
1285
1286            // Finalize
1287            this.view = new MutableView(kview);
1288            this.term = RaftKVDatabase.this.getLastLogTerm();
1289            this.index = RaftKVDatabase.this.getLastLogIndex();
1290        }

1315    /**
1316     * Request service to be invoked after the current service (if any) completes.
1317     *
1318     * <p>
1319     * If {@code service} has an associated {@link Role}, and that {@link Role} is no longer active
1320     * when the service is handled, nothing will be done.
1321     *
1322     * @param service the service to perform
1323     */
1324    private void requestService(Service service) {
1325        assert Thread.holdsLock(this);
1326        if (!this.pendingService.add(service))
1327            return;
1328        if (this.performingService)
1329            return;
1330        try {
1331            this.executor.submit(new ErrorLoggingRunnable() {
1332                @Override
1333                protected void doRun() {
1334                    RaftKVDatabase.this.handlePendingService();
1335                }
1336            });
1337        } catch (RejectedExecutionException e) {
1338            if (!this.shuttingDown)
1339                this.warn("executor task rejected, skipping", e);
1340        }
1341    }

1343    // Performs pending service requests (do not invoke directly)
1344    private synchronized void handlePendingService() {
1345
1346        // Sanity check
1347        assert this.checkState();
1348        assert Thread.currentThread() instanceof ServiceThread;
1349        if (this.role == null)
1350            return;
1351
1352        // While there is work to do, do it
1353        this.performingService = true;
1354        try {
1355            while (!this.pendingService.isEmpty()) {
1356                final Iterator<Service> i = this.pendingService.iterator();
1357                final Service service = i.next();
1358                i.remove();
1359                assert service.getRole() == null || service.getRole() == this.role;
1360                if (this.log.isTraceEnabled())
1361                    this.trace("SERVICE [" + service + "] in " + this.role);
1362                service.run();
1363            }
1364        } finally {
1365            this.performingService = false;
1366        }
1367    }

1398    /**
1399     * One shot timer that {@linkplain #requestService requests} a {@link Service} on expiration.
1400     *
1401     * <p>
1402     * This implementation avoids any race conditions between scheduling, firing, and cancelling.
1403     */
1404    class Timer {
1405
1406        private final Logger log = RaftKVDatabase.this.log;
1407        private final String name;
1408        private final Service service;
1409        private ScheduledFuture<?> future;
1410        private PendingTimeout pendingTimeout;                  // non-null IFF timeout has not been handled yet
1411        private Timestamp timeoutDeadline;
1412
1413        public Timer(String name, Service service) {
1414            this.name = name;
1415            this.service = service;
1416        }
1417
1418        /**
1419         * Stop timer if running.
1420         *
1421         * @throws IllegalStateException if the lock object is not locked
1422         */
1423        public void cancel() {
1424
1425            // Sanity check
1426            assert Thread.holdsLock(RaftKVDatabase.this);
1427
1428            // Cancel existing timer, if any
1429            if (this.future != null) {
1430                this.future.cancel(false);
1431                this.future = null;
1432            }
1433
1434            // Ensure the previously scheduled action does nothing if case we lose the cancel() race condition
1435            this.pendingTimeout = null;
1436            this.timeoutDeadline = null;
1437        }
1438
1439        /**
1440         * (Re)schedule this timer. Discards any previously scheduled timeout.
1441         *
1442         * @param delay delay before expiration in milliseonds
1443         * @return true if restarted, false if executor rejected the task
1444         * @throws IllegalStateException if the lock object is not locked
1445         */
1446        public void timeoutAfter(int delay) {
1447
1448            // Sanity check
1449            assert Thread.holdsLock(RaftKVDatabase.this);
1450            Preconditions.checkArgument(delay >= 0, "delay < 0");
1451
1452            // Cancel existing timeout action, if any
1453            this.cancel();
1454            assert this.future == null;
1455            assert this.pendingTimeout == null;
1456            assert this.timeoutDeadline == null;
1457
1458            // Reschedule new timeout action
1459            this.timeoutDeadline = new Timestamp().offset(delay);
1460            if (this.log.isTraceEnabled()) {
1461                RaftKVDatabase.this.trace("rescheduling " + this.name + " for " + this.timeoutDeadline
1462                  + " (" + delay + "ms from now)");
1463            }
1464            this.pendingTimeout = new PendingTimeout();
1465            try {
1466                this.future = RaftKVDatabase.this.executor.schedule(this.pendingTimeout, delay, TimeUnit.MILLISECONDS);
1467            } catch (RejectedExecutionException e) {
1468                if (!RaftKVDatabase.this.shuttingDown)
1469                    RaftKVDatabase.this.warn("can't restart timer", e);
1470            }
1471        }
1472
1473        /**
1474         * Force timer to expire immediately.
1475         */
1476        public void timeoutNow() {
1477            this.timeoutAfter(0);
1478        }
1479
1480        /**
1481         * Determine if this timer has expired and requires service handling, and reset it if so.
1482         *
1483         * <p>
1484         * If this timer is not running, has not yet expired, or has previously expired and this method was already
1485         * thereafter invoked, false is returned. Otherwise, true is returned, this timer is {@link #cancel}ed (if necessary),
1486         * and the caller is expected to handle the implied service need.
1487         *
1488         * @return true if timer needs handling, false otherwise
1489         */
1490        public boolean pollForTimeout() {
1491
1492            // Sanity check
1493            assert Thread.holdsLock(RaftKVDatabase.this);
1494
1495            // Has timer expired?
1496            if (this.pendingTimeout == null || !this.timeoutDeadline.hasOccurred())
1497                return false;
1498
1499            // Yes, timer requires service
1500            if (Timer.this.log.isTraceEnabled())
1501                RaftKVDatabase.this.trace(Timer.this.name + " expired " + -this.timeoutDeadline.offsetFromNow() + "ms ago");
1502            this.cancel();
1503            return true;
1504        }
1505
1506        /**
1507         * Determine if this timer is running, i.e., will expire or has expired but
1508         * {@link #pollForTimeout} has not been invoked yet.
1509         */
1510        public boolean isRunning() {
1511            return this.pendingTimeout != null;
1512        }
1513
1514        private class PendingTimeout extends ErrorLoggingRunnable {
1515
1516            @Override
1517            protected void doRun() {
1518                synchronized (RaftKVDatabase.this) {
1519
1520                    // Avoid cancel() race condition
1521                    if (Timer.this.pendingTimeout != this)
1522                        return;
1523
1524                    // Trigger service
1525                    RaftKVDatabase.this.requestService(Timer.this.service);
1526                }
1527            }
1528        }
1529    }

1534     * Reset the persisted state machine to its initial state.
1535     */

1537
1538        // Sanity check
1539        assert Thread.holdsLock(this);
1540        if (this.log.isDebugEnabled())
1541            this.debug("resetting state machine");
1542
1543        // Set invalid values while we make non-atomic changes, in case we crash in the middle
1544        if (!this.recordLastApplied(-1, -1, null))
1545            return false;
1546
1547        // Delete all key/value pairs
1548        this.kv.removeRange(STATE_MACHINE_PREFIX, ByteUtil.getKeyAfterPrefix(STATE_MACHINE_PREFIX));
1549        assert !this.kv.getRange(STATE_MACHINE_PREFIX, ByteUtil.getKeyAfterPrefix(STATE_MACHINE_PREFIX), false).hasNext();

1557            }
1558        }
1559
1560        // Optionally finish intialization
1561        if (initialize && !this.recordLastApplied(0, 0, null))
1562            return false;

1565        if (this.log.isDebugEnabled())
1566            this.debug("done resetting state machine");

1572     */

1574
1575        // Sanity check
1576        assert Thread.holdsLock(this);
1577        if (this.log.isTraceEnabled())
1578            this.trace("updating state machine last applied to " + index + "t" + term + " with config " + config);
1579        if (config == null)
1580            config = new HashMap<String, String>(0);
1581
1582        // Prepare updates
1583        final Writes writes = new Writes();
1584        writes.getPuts().put(LAST_APPLIED_TERM_KEY, LongEncoder.encode(term));
1585        writes.getPuts().put(LAST_APPLIED_INDEX_KEY, LongEncoder.encode(index));
1586        writes.getPuts().put(LAST_APPLIED_CONFIG_KEY, this.encodeConfig(config));
1587
1588        // Update persistent store
1589        try {
1590            this.kv.mutate(writes, true);
1591        } catch (Exception e) {
1592            this.error("error updating key/value store term/index to " + index + "t" + term, e);
1593            return false;
1594        }
1595
1596        // Update in-memory copy
1597        this.lastAppliedTerm = Math.max(term, 0);
1598        this.lastAppliedIndex = Math.max(index, 0);
1599        this.lastAppliedConfig = config;
1600        this.commitIndex = this.lastAppliedIndex;
1601        this.currentConfig = this.buildCurrentConfig();
1602        if (term >= 0 && index >= 0)
1603            this.info("new cluster configuration: " + this.currentConfig);
1604        return true;

1898    private boolean sendMessage(Message msg) {
1899
1900        // Sanity check
1901        assert Thread.holdsLock(this);
1902
1903        // Get peer's address; if unknown, use the return address of the message being processed (if any)
1904        final String peer = msg.getRecipientId();
1905        String address = this.currentConfig.get(peer);
1906        if (address == null)
1907            address = this.returnAddress;
1908        if (address == null) {
1909            this.warn("can't send " + msg + " to unknown peer \"" + peer + "\"");
1910            return false;
1911        }
1912
1913        // Send message
1914        if (this.log.isTraceEnabled())
1915            this.trace("XMIT " + msg + " to " + address);
1916        if (this.network.send(address, msg.encode())) {
1917            this.transmitting.add(address);
1918            return true;
1919        }
1920
1921        // Transmit failed
1922        this.warn("transmit of " + msg + " to \"" + peer + "\" failed locally");
1923        return false;
1924    }

1926    private synchronized void receiveMessage(String address, Message msg) {
1927
1928        // Sanity check
1929        assert Thread.holdsLock(this);
1930        assert this.checkState();
1931        if (this.role == null) {
1932            if (this.log.isDebugEnabled())
1933                this.debug("rec'd " + msg + " rec'd in shutdown state; ignoring");
1934            return;
1935        }
1936
1937        // Sanity check cluster ID
1938        if (msg.getClusterId() == 0) {
1939            this.warn("rec'd " + msg + " with zero cluster ID from " + address + "; ignoring");
1940            return;
1941        }
1942        if (this.clusterId != 0 && msg.getClusterId() != this.clusterId) {
1943            this.warn("rec'd " + msg + " with foreign cluster ID "
1944              + String.format("0x%08x", msg.getClusterId()) + " != " + String.format("0x%08x", this.clusterId) + "; ignoring");
1945            return;
1946        }
1947
1948        // Sanity check sender
1949        final String peer = msg.getSenderId();
1950        if (peer.equals(this.identity)) {
1951            this.warn("rec'd " + msg + " from myself (\"" + peer + "\", address " + address + "); ignoring");
1952            return;
1953        }
1954
1955        // Sanity check recipient
1956        final String dest = msg.getRecipientId();
1957        if (!dest.equals(this.identity)) {
1958            this.warn("rec'd misdirected " + msg + " intended for \"" + dest + "\" from " + address + "; ignoring");
1959            return;
1960        }
1961
1962        // Is sender's term too low? Ignore it
1963        if (msg.getTerm() < this.currentTerm) {
1964            if (this.log.isDebugEnabled()) {
1965                this.debug("rec'd " + msg + " with term " + msg.getTerm() + " < " + this.currentTerm
1966                  + " from \"" + peer + "\" at " + address + ", ignoring");
1967            }
1968            return;
1969        }
1970
1971        // Is my term too low? If so update and revert to follower
1972        if (msg.getTerm() > this.currentTerm) {
1973
1974            // First check with current role; in some special cases we ignore this
1975            if (!this.role.mayAdvanceCurrentTerm(msg)) {
1976                if (this.log.isTraceEnabled()) {
1977                    this.trace("rec'd " + msg + " with term " + msg.getTerm() + " > " + this.currentTerm + " from \""
1978                      + peer + "\" but current role says to ignore it");
1979                }
1980                return;
1981            }
1982
1983            // Revert to follower
1984            if (this.log.isDebugEnabled()) {
1985                this.debug("rec'd " + msg.getClass().getSimpleName() + " with term " + msg.getTerm() + " > "
1986                  + this.currentTerm + " from \"" + peer + "\", updating term and "
1987                  + (this.role instanceof FollowerRole ? "remaining a" : "reverting to") + " follower");
1988            }
1989            if (!this.advanceTerm(msg.getTerm()))
1990                return;
1991            this.changeRole(msg.isLeaderMessage() ? new FollowerRole(this, peer, address) : new FollowerRole(this));
1992        }
1993
1994        // Debug
1995        if (this.log.isTraceEnabled())
1996            this.trace("RECV " + msg + " in " + this.role + " from " + address);
1997
1998        // Handle message
1999        this.returnAddress = address;
2000        try {
2001            msg.visit(new MessageSwitch() {
2002                @Override
2003                public void caseAppendRequest(AppendRequest msg) {
2004                    RaftKVDatabase.this.role.caseAppendRequest(msg);
2005                }
2006                @Override
2007                public void caseAppendResponse(AppendResponse msg) {
2008                    RaftKVDatabase.this.role.caseAppendResponse(msg);
2009                }
2010                @Override
2011                public void caseCommitRequest(CommitRequest msg) {
2012                    RaftKVDatabase.this.role.caseCommitRequest(msg);
2013                }
2014                @Override
2015                public void caseCommitResponse(CommitResponse msg) {
2016                    RaftKVDatabase.this.role.caseCommitResponse(msg);
2017                }
2018                @Override
2019                public void caseGrantVote(GrantVote msg) {
2020                    RaftKVDatabase.this.role.caseGrantVote(msg);
2021                }
2022                @Override
2023                public void caseInstallSnapshot(InstallSnapshot msg) {
2024                    RaftKVDatabase.this.role.caseInstallSnapshot(msg);
2025                }
2026                @Override
2027                public void caseRequestVote(RequestVote msg) {
2028                    RaftKVDatabase.this.role.caseRequestVote(msg);
2029                }
2030            });
2031        } finally {
2032            this.returnAddress = null;
2033        }
2034    }

2402        /**
2403         * Check a transaction waiting for its log entry to be committed (in the {@link TxState#COMMIT_WAITING} state).
2404         *
2405         * <p>
2406         * This should be invoked:
2407         * <ul>
2408         *  <li>After changing roles</li>
2409         *  <li>After a transaction has entered the {@link TxState#COMMIT_WAITING} state</li>
2410         *  <li>After advancing my {@code commitIndex} (as leader or follower)</li>
2411         *  <li>After receiving an updated {@linkplain AppendResponse#getLeaderLeaseTimeout leader lease timeout}
2412         *      (in {@link FollowerRole})</li>
2413         * </ul>
2414         *
2415         * @param tx the transaction
2416         * @throws KVTransactionException if an error occurs
2417         */
2418        private void checkWaitingTransaction(RaftKVTransaction tx) {
2419
2420            // Handle the case the transaction's committed log index has already been applied to the state machine
2421            final long commitIndex = tx.getCommitIndex();
2422            if (commitIndex < this.raft.lastAppliedIndex) {
2423
2424                // This can happen if we lose contact and by the time we're back the log entry has
2425                // already been applied to the state machine on some leader and that leader sent
2426                // use an InstallSnapshot message. We don't know whether it actually got committed
2427                // or not, so the transaction must be retried.
2428                throw new RetryTransactionException(tx, "committed log entry was missed");
2429            }
2430
2431            // Has the transaction's log entry been received and committed yet?
2432            if (commitIndex > this.raft.commitIndex)
2433                return;
2434
2435            // Verify the term of the committed log entry; if not what we expect, the log entry was overwritten by a new leader
2436            final long commitTerm = tx.getCommitTerm();
2437            if (this.raft.getLogTermAtIndex(commitIndex) != commitTerm)
2438                throw new RetryTransactionException(tx, "leader was deposed during commit and transaction's log entry overwritten");
2439
2440            // Check with subclass
2441            if (!this.mayCommit(tx))
2442                return;
2443
2444            // Transaction is officially committed now
2445            if (this.log.isTraceEnabled())
2446                this.trace("commit successful for " + tx + " (commit index " + this.raft.commitIndex + " >= " + commitIndex + ")");
2447            this.raft.succeed(tx);
2448        }

2735        @Override
2736        boolean mayApplyLogEntry(LogEntry logEntry) {
2737
2738            // Are we running out of memory, or keeping around too many log entries? If so, go ahead.
2739            final long logEntryMemoryUsage = this.raft.getUnappliedLogMemoryUsage();
2740            if (logEntryMemoryUsage > this.raft.maxAppliedLogMemory
2741              || this.raft.raftLog.size() > MAX_UNAPPLIED_LOG_ENTRIES) {
2742                if (this.log.isTraceEnabled()) {
2743                    this.trace("allowing log entry " + logEntry + " to be applied because memory usage "
2744                      + logEntryMemoryUsage + " > " + this.raft.maxAppliedLogMemory + " and/or log length "
2745                      + this.raft.raftLog.size() + " > " + MAX_UNAPPLIED_LOG_ENTRIES);
2746                }
2747                return true;
2748            }
2749
2750            // Try to keep log entries around for a minimum amount of time to facilitate long-running transactions on followers
2751            if (logEntry.getAge() < this.raft.maxTransactionDuration) {
2752                if (this.log.isTraceEnabled()) {
2753                    this.trace("delaying application of " + logEntry + " because it has age "
2754                      + logEntry.getAge() + "ms < " + this.raft.maxTransactionDuration + "ms");
2755                }
2756                return false;
2757            }
2758
2759            // If any snapshots are in progress, we don't want to apply any log entries with index greater than the snapshot's
2760            // index, because then we'd "lose" the ability to update the follower with that log entry, and as a result just have
2761            // to send a snapshot again. However, we impose a limit on how long we'll wait for a slow follower.
2762            for (Follower follower : this.followerMap.values()) {
2763                final SnapshotTransmit snapshotTransmit = follower.getSnapshotTransmit();
2764                if (snapshotTransmit == null)
2765                    continue;
2766                if (snapshotTransmit.getSnapshotIndex() < logEntry.getIndex()
2767                  && snapshotTransmit.getAge() < MAX_SNAPSHOT_TRANSMIT_AGE) {
2768                    if (this.log.isTraceEnabled()) {
2769                        this.trace("delaying application of " + logEntry + " because of in-progress snapshot install of "
2770                          + snapshotTransmit.getSnapshotIndex() + "t" + snapshotTransmit.getSnapshotTerm()
2771                          + " to " + follower);
2772                    }
2773                    return false;
2774                }
2775            }
2776
2777            // If some follower does not yet have the log entry, wait for them to get it (up to some maximum time)
2778            if (logEntry.getAge() < MAX_SLOW_FOLLOWER_APPLY_DELAY_HEARTBEATS * this.raft.heartbeatTimeout) {
2779                for (Follower follower : this.followerMap.values()) {
2780                    if (follower.getMatchIndex() < logEntry.getIndex()) {
2781                        if (this.log.isTraceEnabled()) {
2782                            this.trace("delaying application of " + logEntry + " (age " + logEntry.getAge()
2783                              + " < " + (MAX_SLOW_FOLLOWER_APPLY_DELAY_HEARTBEATS * this.raft.heartbeatTimeout)
2784                              + ") because of slow " + follower);
2785                        }
2786                        return false;
2787                    }
2788                }
2789            }
2790
2791            // OK
2792            return true;
2793        }

2803         * Update my {@code commitIndex} based on followers' {@code matchIndex}'s.
2804         *
2805         * <p>
2806         * This should be invoked:
2807         * <ul>
2808         *  <li>After any log entry has been added to the log, if we have zero followers</li>
2809         *  <li>After a log entry that contains a configuration change has been added to the log</li>
2810         *  <li>After a follower's {@linkplain Follower#getMatchIndex match index} has advanced</li>
2811         * </ul>
2812         */
2813        private void updateLeaderCommitIndex() {
2814
2815            // Find highest index for which a majority of cluster members have ack'd the corresponding log entry from my term
2816            final int totalCount = this.raft.currentConfig.size();                          // total possible nodes
2817            final int requiredCount = totalCount / 2 + 1;                                   // require a majority
2818            final int startingCount = this.raft.isClusterMember() ? 1 : 0;                  // count myself, if member
2819            long maxCommitIndex = this.raft.commitIndex;
2820            int commitCount = -1;
2821            for (long index = this.raft.commitIndex + 1; index <= this.raft.getLastLogIndex(); index++) {
2822
2823                // Count the number of nodes (possibly including myself) that have a copy of the log entry at index
2824                int count = startingCount;
2825                for (Follower follower : this.followerMap.values()) {                       // count followers who are members
2826                    if (follower.getMatchIndex() >= index && this.raft.isClusterMember(follower.getIdentity()))
2827                        count++;
2828                }
2829
2830                // The log entry term must match my current term (exception: unless every node has it)
2831                final long term = this.raft.getLogTermAtIndex(index);
2832                if (count < totalCount && term != this.raft.currentTerm)
2833                    continue;
2834
2835                // Do a majority of cluster nodes have this log entry?
2836                if (count < requiredCount) {
2837                    if (term >= this.raft.currentTerm)                                      // there's no point in going further
2838                        break;
2839                    continue;                                                               // a later term log entry might work
2840                }
2841
2842                // We have a winner
2843                maxCommitIndex = index;
2844                commitCount = count;
2845            }
2846
2847            // Update commit index if it advanced
2848            if (maxCommitIndex > this.raft.commitIndex) {
2849
2850                // Update index
2851                if (this.log.isDebugEnabled()) {
2852                    this.debug("advancing commit index from " + this.raft.commitIndex + " -> " + maxCommitIndex + " based on "
2853                      + commitCount + "/" + totalCount + " nodes having received " + this.raft.getLogEntryAtIndex(maxCommitIndex));
2854                }
2855                this.raft.commitIndex = maxCommitIndex;
2856
2857                // Perform various service
2858                this.raft.requestService(this.checkReadyTransactionsService);
2859                this.raft.requestService(this.checkWaitingTransactionsService);
2860                this.raft.requestService(this.applyCommittedLogEntriesService);
2861
2862                // Notify all (up-to-date) followers with the updated leaderCommit
2863                this.updateAllSynchronizedFollowersNow();

2865                // If we are no longer a member of the cluster, step down after the most recent config change is committed
2866                if (!this.raft.isClusterMember() && this.raft.commitIndex >= this.findMostRecentConfigChange()) {

2876         *
2877         * <p>
2878         * This should be invoked:
2879         * <ul>
2880         *  <li>After a follower has replied with an {@link AppendResponse} containing a newer
2881         *      {@linkplain AppendResponse#getLeaderTimestamp leader timestamp} than before</li>
2882         * </ul>

2887            final int numFollowers = this.followerMap.size();
2888            if (numFollowers == 0)
2889                return;

2892            final Timestamp[] leaderTimestamps = new Timestamp[this.raft.currentConfig.size()];
2893            int index = 0;
2894            if (this.raft.isClusterMember())
2895                leaderTimestamps[index++] = new Timestamp();                        // this represents my own vote
2896            for (Follower follower : this.followerMap.values()) {
2897                if (this.raft.isClusterMember(follower.getIdentity()))

2899            }

2905            //  # nodes    timestamps
2906            //  -------    ----------
2907            //     5       [ ][ ][x][x][x]        3/5 x's make a majority at index (5 - 1)/2 = 2
2908            //     6       [ ][ ][x][x][x][x]     4/6 x's make a majority at index (6 - 1)/2 = 2

2911            // We then add the minimum election timeout, then subtract a little for clock drift.
2912            //

2914              .offset((int)(this.raft.minElectionTimeout * (1.0f - MAX_CLOCK_DRIFT) - 1));

2924                for (Follower follower : this.followerMap.values()) {

2926                    if (!timeouts.isEmpty()) {

2930                }

4401            }

4404            if (msg.isLastChunk()) {

4406                if (this.log.isDebugEnabled()) {
4407                    this.debug("snapshot install from \"" + msg.getSenderId() + "\" of "
4408                      + index + "t" + term + " with config " + snapshotConfig + " complete");
4409                }
4410                this.snapshotReceive = null;
4411                if (!this.raft.recordLastApplied(term, index, snapshotConfig))

4438                }
4439                return;
4440            }
4441
4442            // Persist our vote for this peer (if not already persisted)
4443            if (this.votedFor == null) {

4451            }

4453            // Send reply
