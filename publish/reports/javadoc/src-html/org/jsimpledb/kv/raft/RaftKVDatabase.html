072/**
073 * A distributed {@link org.jsimpledb.kv.KVDatabase} based on the Raft consensus algorithm.
074 *
075 * <p>
076 * Raft defines a distributed consensus algorithm for maintaining a shared state machine.
077 * Each Raft node maintains a complete copy of the state machine. Cluster nodes elect a
078 * leader who collects and distributes updates and provides for consistent reads.
079 * As long as as a node is part of a majority, the state machine is fully operational.
080 *
081 * <p>
082 * {@link RaftKVDatabase} turns this into a transactional key/value database with linearizable ACID semantics as follows:
083 *  <ul>
084 *  <li>The Raft state machine is the key/value store data.</li>
085 *  <li>Unapplied log entries are stored on disk as serialized mutations, and also cached in memory.</li>
086 *  <li>On leaders only, committed log entries are not applied immediately; instead they kept around for time at least
087 *      <i>T<sub>max</sub></i> since creation, where <i>T<sub>max</sub></i> is the maximum supported transaction duration.
088 *      This caching is subject to <i>M<sub>max</sub></i>, which limits the total memory used; if reached, these log entries
089 *      are applied early.</li>
090 *  <li>Concurrent transactions are supported through a simple optimistic locking MVCC scheme (same as used by
091 *      {@link org.jsimpledb.kv.mvcc.SnapshotKVDatabase}):
092 *      <ul>
093 *      <li>Transactions execute locally until commit time, using a {@link org.jsimpledb.kv.mvcc.MutableView} to collect mutations.
094 *          The {@link org.jsimpledb.kv.mvcc.MutableView} is based on the local node's most recent log entry
095 *          (whether committed or not); this is called the <i>base term and index</i> for the transaction.</li>
096 *      <li>On commit, the transaction's {@link org.jsimpledb.kv.mvcc.Reads}, {@link org.jsimpledb.kv.mvcc.Writes},
097 *          base index and term, and any config change are {@linkplain CommitRequest sent} to the leader.</li>
098 *      <li>The leader confirms that the log entry corresponding to the transaction's base index is either not yet applied,
099 *          or was its most recently applied log entry. If this is not the case, then the transaction's base log entry
100 *          is too old (older than <i>T<sub>max</sub></i>, or was applied and discarded early due to memory pressure),
101 *          and so the transaction is rejected with a {@link RetryTransactionException}.
102 *      <li>The leader verifies that the the log entry term matches the transaction's base term; if not, the base log entry
103 *          has been overwritten, and the transaction is rejected with a {@link RetryTransactionException}.
104 *      <li>The leader confirms that the {@link Writes} associated with log entries after the transaction's base log entry
105 *          do not create {@linkplain org.jsimpledb.kv.mvcc.Reads#isConflict conflicts} when compared against the transaction's
106 *          {@link org.jsimpledb.kv.mvcc.Reads}. If so, the transaction is rejected with a {@link RetryTransactionException}.</li>
107 *      <li>The leader adds a new log entry consisting of the transaction's {@link Writes} (and any config change) to its log.
108 *          The associated term and index become the transaction's <i>commit term and index</i>; the leader then
109 *          {@linkplain CommitResponse replies} to the follower with this information.</li>
110 *      <li>If/when the follower sees a <i>committed</i> (in the Raft sense) log entry appear in its log matching the
111 *          transaction's commit term and index, then the transaction is complete.</li>
112 *      <li>As an optimization, when the leader sends a log entry to the same follower who committed the corresponding
113 *          transaction in the first place, only the transaction ID is sent, because the follower already has the data.</li>
114 *      </ul>
115 *  </li>
116 *  <li>For transactions occurring on a leader, the logic is similar except of course no network communication occurs.</li>
117 *  <li>For read-only transactions, the leader does not create a new log entry; instead, the transaction's commit
118 *      term and index are set to the base term and index, and the leader also calculates its current "leader lease timeout",
119 *      which is the earliest time at which it is possible for another leader to be elected.
120 *      This is calculated as the time in the past at which the leader sent {@link AppendRequest}'s to a majority of followers
121 *      who have since responded, plus the {@linkplain #setMinElectionTimeout minimum election timeout}, minus a small adjustment
122 *      for possible clock drift (this assumes all nodes have the same minimum election timeout configured). If the current
123 *      time is prior to the leader lease timeout, the transaction may be committed as soon as log entry corresponding to the
124 *      commit term and index is committed (it may already be); otherwise, the current time is returned to the follower
125 *      as minimum required leader lease timeout before the transaction may be committed.</li>
126 *  <li>Every {@link AppendRequest} includes the leader's current timestamp and leader lease timeout, so followers can commit
127 *      any waiting read-only transactions. Leaders keep track of which followers are waiting on which leader lease
128 *      timeout values, and when the leader lease timeout advances to allow a follower to commit a transaction, the follower
129 *      is immediately notified.</li>
130 *  <li>Optional weaker consistency guarantees are availble on a per-transaction bases; see
131 *      {@link RaftKVTransaction#setConsistency RaftKVTransaction.setConsistency()}.</li>
132 *  </ul>
133 *
134 * <p><b>Limitations</b></p>
135 *
136 * <ul>
137 *  <li>A transaction's mutations must fit in memory.</li>
138 *  <li>An {@link AtomicKVStore} is required to store local persistent state.</li>
139 *  <li>All nodes must be configured with the same {@linkplain #setMinElectionTimeout minimum election timeout}.
140 *      This guarantees that the leader's lease timeout calculation is valid.</li>
141 *  <li>Due to the optimistic locking approach used, this implementation will perform poorly when there is a high
142 *      rate of conflicting transactions; the result will be many transaction retries.</li>
143 *  <li>Performance will suffer when the amount of data associated with a typical transaction cannot be delivered
144 *      quickly and reliably over the network.</li>
145 * </ul>
146 *
147 * <p>
148 * In general, the algorithm should function correctly under all non-Byzantine conditions. The level of difficultly
149 * the system is experiencing, due to contention, network errors, etc., can be measured in terms of:
150 * <ul>
151 *  <li>The average amount of time it takes to commit a transaction</li>
152 *  <li>The frequency of {@link RetryTransactionException}'s</li>
153 * </ul>
154 *
155 * <p><b>Cluster Configuration</b></p>
156 *
157 * <p>
158 * Instances support dynamic cluster configuration changes at runtime.
159 *
160 * <p>
161 * Initially, all nodes are in an <i>unconfigured</i> state, where nothing has been added to the Raft log yet and no
162 * cluster is defined. Unconfigured nodes are passive: they stay in follower mode (i.e., they will not start elections),
163 * and they disallow local transactions that make any changes other than as described below to create a new cluster.
164 *
165 * <p>
166 * An unconfigured node becomes configured when either:
167 * <ol>
168 *  <li>{@link RaftKVTransaction#configChange RaftKVTransaction.configChange()} is invoked and committed within
169 *      a local transaction, which creates a new single node cluster and commits the first log entry; or</li>
170 *  <li>An {@link AppendRequest} is received from a leader of some existing cluster, in which case the node
171 *      records the cluster ID thereby joining the cluster (see below), and applies the received cluster configuration.</li>
172 * </ol>
173 *
174 * <p>
175 * A node is configured if and only if it has recorded one or more log entries. The very first log entry
176 * always contains the initial cluster configuration (containing only the node that created it, whether local or remote),
177 * so any node that has a non-empty log is configured.
178 *
179 * <p>
180 * Newly created clusters are assigned a random 32-bit cluster ID (option #1 above). This ID is included in all messages sent
181 * over the network, and adopted by unconfigured nodes that join the cluster (via option #2 above). Configured nodes discard
182 * incoming messages containing a cluster ID different from the one they have joined. This prevents data corruption that can
183 * occur if nodes from two different clusters are inadvertently "mixed" together.
184 *
185 * <p>
186 * Once a node joins a cluster with a specific cluster ID, it cannot be reassigned to a different cluster without first
187 * returning it to the unconfigured state; to do that, it must be shut it down and its persistent state deleted.
188 *
189 * <p><b>Configuration Changes</b></p>
190 *
191 * <p>
192 * Once a node is configured, a separate issue is whether the node is <i>included</i> in its own configuration, i.e., whether
193 * the node is a member of its cluster according to the current cluster configuration. A node that is not a member of its
194 * cluster does not count its own vote to determine committed log entries (if a leader), and does not start elections
195 * (if a follower). However, it will accept and respond to incoming {@link AppendRequest}s and {@link RequestVote}s.
196 *
197 * <p>
198 * In addition, leaders follow these rules with respect to configuration changes:
199 * <ul>
200 *  <li>If a leader is removed from a cluster, it remains the leader until the corresponding configuration change
201 *      is committed (not counting its own vote), and then steps down (i.e., reverts to follower).</li>
202 *  <li>If a follower is added to a cluster, the leader immediately starts sending that follower {@link AppendRequest}s.</li>
203 *  <li>If a follower is removed from a cluster, the leader continues to send that follower {@link AppendRequest}s
204 *      until the follower acknowledges receipt of the log entry containing the configuration change.</li>
205 *  <li>Leaders defer configuration changes until they have committed at least one log entry in the current term
206 *      (see <a href="https://groups.google.com/d/msg/raft-dev/t4xj6dJTP6E/d2D9LrWRza8J">this discussion</a>).</li>
207 *  <li>Configuration changes that remove the last node in a cluster are disallowed.</li>
208 *  <li>Only one configuration change may take place at a time.</li>
209 * </ul>
210 *
211 * <p><b>Follower Probes</b></p>
212 *
213 * <p>
214 * This implementation includes a modification to the Raft state machine to avoid unnecessary, disruptive elections
215 * when a node or nodes is disconnected from, and then reconnected to, the majority.
216 *
217 * <p>
218 * When a follower's election timeout fires, before converting into a candidate, the follower is required to verify
219 * communication with a majority of the cluster using {@linkplain PingRequest} messages. Only when the follower has
220 * successfully done so may it become a candidate.  While in this intermediate "probing" mode, the follower responds
221 * normally to incoming messages. In particular, if the follower receives a valid {@link AppendRequest} from the leader, it
222 * reverts back to normal operation.
223 *
224 * <p>
225 * This behavior is optional, but enabled by default (see {@link #setFollowerProbingEnabled setFollowerProbingEnabled()});
226 *
227 * <p><b>Key Watches</b></p>
228 *
229 * <p>
230 * {@linkplain RaftKVTransaction#watchKey Key watches} are supported.
231 *
232 * @see <a href="https://raftconsensus.github.io/">The Raft Consensus Algorithm</a>
233 */

794    @Override
795    @PostConstruct
796    public synchronized void start() {
797
798        // Sanity check
799        assert this.checkState();
800        if (this.role != null)
801            return;
802        Preconditions.checkState(!this.shuttingDown, "shutdown in progress");
803        Preconditions.checkState(this.logDir != null, "no Raft log directory configured");
804        Preconditions.checkState(this.kv != null, "no Raft local persistence key/value store configured");
805        Preconditions.checkState(this.network != null, "no Raft network configured");
806        Preconditions.checkState(this.minElectionTimeout <= this.maxElectionTimeout, "minElectionTimeout > maxElectionTimeout");
807        Preconditions.checkState(this.heartbeatTimeout < this.minElectionTimeout, "heartbeatTimeout >= minElectionTimeout");
808        Preconditions.checkState(this.identity != null, "no Raft identity configured");
809
810        // Log
811        if (this.log.isDebugEnabled())
812            this.debug("starting " + this.getClass().getName() + " in directory " + this.logDir);
813
814        // Start up local database
815        boolean success = false;
816        try {
817
818            // Create/verify log directory
819            if (!this.logDir.exists() && !this.logDir.isDirectory())
820                throw new IOException("failed to create directory `" + this.logDir + "'");
821            if (!this.logDir.isDirectory())
822                throw new IOException("file `" + this.logDir + "' is not a directory");
823
824            // Start k/v store
825            this.kv.start();
826
827            // Open directory containing log entry files so we have a way to fsync() it
828            assert this.logDirChannel == null;
829            this.logDirChannel = FileChannel.open(this.logDir.toPath(), StandardOpenOption.READ);
830
831            // Create randomizer
832            assert this.random == null;
833            this.random = new SecureRandom();
834
835            // Start up service executor thread
836            assert this.serviceExecutor == null;
837            this.serviceExecutor = Executors.newSingleThreadScheduledExecutor(new ThreadFactory() {
838                @Override
839                public Thread newThread(Runnable action) {
840                    final Thread thread = new Thread(action);
841                    thread.setName(RaftKVDatabase.this + " Service");
842                    return thread;
843                }
844            });
845
846            // Start network
847            this.network.start(new Network.Handler() {
848                @Override
849                public void handle(String sender, ByteBuffer buf) {
850                    RaftKVDatabase.this.handle(sender, buf);
851                }
852
853                @Override
854                public void outputQueueEmpty(String address) {
855                    RaftKVDatabase.this.outputQueueEmpty(address);
856                }
857            });
858
859            // Reload persistent raft info
860            this.clusterId = (int)this.decodeLong(CLUSTER_ID_KEY, 0);
861            this.currentTerm = this.decodeLong(CURRENT_TERM_KEY, 0);
862            final String votedFor = this.decodeString(VOTED_FOR_KEY, null);
863            this.lastAppliedTerm = this.decodeLong(LAST_APPLIED_TERM_KEY, 0);
864            this.lastAppliedIndex = this.decodeLong(LAST_APPLIED_INDEX_KEY, 0);
865            this.lastAppliedConfig = this.decodeConfig(LAST_APPLIED_CONFIG_KEY);
866            this.flipflop = this.decodeBoolean(FLIP_FLOP_KEY);
867            this.currentConfig = this.buildCurrentConfig();
868
869            // If we crashed part way through a snapshot install, recover by discarding partial install
870            if (this.discardFlipFloppedStateMachine() && this.log.isDebugEnabled())
871                this.debug("detected partially applied snapshot install, discarding");
872
873            // Initialize commit index and key watch index
874            this.commitIndex = this.lastAppliedIndex;
875            this.keyWatchIndex = this.commitIndex;
876
877            // Reload outstanding log entries from disk
878            this.loadLog();
879
880            // Show recovered state
881            if (this.log.isDebugEnabled()) {
882                this.debug("recovered Raft state:"
883                  + "\n  clusterId=" + (this.clusterId != 0 ? String.format("0x%08x", this.clusterId) : "none")
884                  + "\n  currentTerm=" + this.currentTerm
885                  + "\n  lastApplied=" + this.lastAppliedIndex + "t" + this.lastAppliedTerm
886                  + "\n  lastAppliedConfig=" + this.lastAppliedConfig
887                  + "\n  currentConfig=" + this.currentConfig
888                  + "\n  votedFor=" + (votedFor != null ? "\"" + votedFor + "\"" : "nobody")
889                  + "\n  log=" + this.raftLog);
890            }
891
892            // Validate recovered state
893            if (this.isConfigured()) {
894                Preconditions.checkArgument(this.clusterId != 0);
895                Preconditions.checkArgument(this.currentTerm > 0);
896                Preconditions.checkArgument(this.getLastLogTerm() > 0);
897                Preconditions.checkArgument(this.getLastLogIndex() > 0);
898                Preconditions.checkArgument(!this.currentConfig.isEmpty());
899            } else {
900                Preconditions.checkArgument(this.lastAppliedTerm == 0);
901                Preconditions.checkArgument(this.lastAppliedIndex == 0);
902                Preconditions.checkArgument(this.getLastLogTerm() == 0);
903                Preconditions.checkArgument(this.getLastLogIndex() == 0);
904                Preconditions.checkArgument(this.currentConfig.isEmpty());
905                Preconditions.checkArgument(this.raftLog.isEmpty());
906            }
907
908            // Start as follower (with unknown leader)
909            this.changeRole(new FollowerRole(this, null, null, votedFor));
910
911            // Done
912            this.info("successfully started " + this + " in directory " + this.logDir);
913            success = true;
914        } catch (IOException e) {
915            throw new RuntimeException("error starting up database", e);
916        } finally {
917            if (!success)
918                this.cleanup();
919        }
920
921        // Sanity check
922        assert this.checkState();
923    }

925    @Override
926    @PreDestroy
927    public void stop() {
928
929        // Set flag to prevent new transactions
930        synchronized (this) {
931
932            // Sanity check
933            assert this.checkState();
934            if (this.role == null || this.shuttingDown)
935                return;
936
937            // Set shutting down flag
938            this.info("starting shutdown of " + this);
939            this.shuttingDown = true;
940
941            // Fail all remaining open transactions
942            for (RaftKVTransaction tx : new ArrayList<RaftKVTransaction>(this.openTransactions.values())) {
943                switch (tx.getState()) {
944                case EXECUTING:
945                    tx.rollback();
946                    break;
947                case COMMIT_READY:
948                case COMMIT_WAITING:
949                    this.fail(tx, new KVTransactionException(tx, "database shutdown"));
950                    break;
951                case COMPLETED:
952                    break;
953                default:
954                    assert false;
955                    break;
956                }
957            }
958
959            // Sleep while we wait for transactions to clean themselves up
960            boolean done = false;
961            try {
962                done = TimedWait.wait(this, 5000, new org.dellroad.stuff.java.Predicate() {
963                    @Override
964                    public boolean test() {
965                        return RaftKVDatabase.this.openTransactions.isEmpty();
966                    }
967                });
968            } catch (InterruptedException e) {
969                Thread.currentThread().interrupt();
970            }
971            if (!done)
972                this.warn("open transactions not cleaned up during shutdown");
973        }
974
975        // Shut down the service executor and wait for pending tasks to finish
976        this.serviceExecutor.shutdownNow();
977        try {
978            this.serviceExecutor.awaitTermination(1000, TimeUnit.MILLISECONDS);
979        } catch (InterruptedException e) {
980            Thread.currentThread().interrupt();
981        }
982
983        // Final cleanup
984        synchronized (this) {
985            this.serviceExecutor = null;
986            this.cleanup();
987        }
988
989        // Done
990        this.info("completed shutdown of " + this);
991    }

993    private void cleanup() {
994        assert Thread.holdsLock(this);
995        assert this.openTransactions.isEmpty();
996        if (this.role != null) {
997            this.role.shutdown();
998            this.role = null;
999        }
1000        if (this.serviceExecutor != null) {
1001            this.serviceExecutor.shutdownNow();

1003                this.serviceExecutor.awaitTermination(1000, TimeUnit.MILLISECONDS);
1004            } catch (InterruptedException e) {
1005                Thread.currentThread().interrupt();
1006            }
1007            this.serviceExecutor = null;
1008        }
1009        this.kv.stop();
1010        Util.closeIfPossible(this.logDirChannel);
1011        this.logDirChannel = null;
1012        this.raftLog.clear();
1013        this.random = null;
1014        this.network.stop();
1015        this.currentTerm = 0;

1018        this.clusterId = 0;

1020        this.lastAppliedIndex = 0;
1021        this.lastAppliedConfig = null;

1024        this.transmitting.clear();
1025        this.pendingService.clear();
1026        this.shuttingDown = false;
1027    }

1029    /**
1030     * Initialize our in-memory state from the persistent state reloaded from disk.
1031     * This is invoked on initial startup.
1032     */
1033    private void loadLog() throws IOException {
1034
1035        // Sanity check
1036        assert Thread.holdsLock(this);
1037        assert this.raftLog.isEmpty();
1038
1039        // Scan for log entry files
1040        this.raftLog.clear();
1041        for (File file : this.logDir.listFiles()) {
1042
1043            // Ignore sub-directories (typically owned by the underlying k/v store)
1044            if (file.isDirectory())
1045                continue;
1046
1047            // Is this a log entry file?
1048            if (LogEntry.LOG_FILE_PATTERN.matcher(file.getName()).matches()) {
1049                if (this.log.isDebugEnabled())
1050                    this.debug("recovering log file " + file.getName());
1051                final LogEntry logEntry = LogEntry.fromFile(file);
1052                this.raftLog.add(logEntry);
1053                continue;
1054            }
1055
1056            // Is this a leftover temporary file?
1057            if (TEMP_FILE_PATTERN.matcher(file.getName()).matches()) {
1058                if (this.log.isDebugEnabled())
1059                    this.debug("deleting leftover temporary file " + file.getName());
1060                if (!file.delete())
1061                    this.error("failed to delete leftover temporary file " + file);
1062                continue;
1063            }
1064
1065            // Unknown
1066            this.warn("ignoring unrecognized file " + file.getName() + " in my log directory");
1067        }
1068
1069        // Verify we have a contiguous range of log entries starting from the snapshot index; discard bogus log files
1070        Collections.sort(this.raftLog, LogEntry.SORT_BY_INDEX);
1071        long lastTermSeen = this.lastAppliedTerm;
1072        long expectedIndex = this.lastAppliedIndex + 1;
1073        for (Iterator<LogEntry> i = this.raftLog.iterator(); i.hasNext(); ) {
1074            final LogEntry logEntry = i.next();
1075            String error = null;
1076            if (logEntry.getTerm() < lastTermSeen)
1077                error = "term " + logEntry.getTerm() + " < last applied term " + lastTermSeen;
1078            else if (logEntry.getIndex() < this.lastAppliedIndex)
1079                error = "index " + logEntry.getIndex() + " < last applied index " + this.lastAppliedIndex;
1080            else if (logEntry.getIndex() != expectedIndex)
1081                error = "index " + logEntry.getIndex() + " != expected index " + expectedIndex;
1082            if (error != null) {
1083                this.warn("deleting bogus log file " + logEntry.getFile().getName() + ": " + error);
1084                if (!logEntry.getFile().delete())
1085                    this.error("failed to delete bogus log file " + logEntry.getFile());
1086                i.remove();
1087            } else {
1088                expectedIndex++;
1089                lastTermSeen = logEntry.getTerm();
1090            }
1091        }
1092        if (this.log.isDebugEnabled()) {
1093            this.debug("recovered " + this.raftLog.size() + " log entries: " + this.raftLog
1094              + " (" + this.getUnappliedLogMemoryUsage() + " total bytes)");
1095        }
1096
1097        // Rebuild current configuration
1098        this.currentConfig = this.buildCurrentConfig();
1099    }

1151    /**
1152     * Create a new transaction with the specified consistency.
1153     *
1154     * <p>
1155     * Transactions that wish to use {@link Consistency#EVENTUAL_COMMITTED} must be created using this method,
1156     * because the log entry on which the transaction is based is determined at creation time.
1157     *
1158     * @param consistency consistency level
1159     * @return newly created transaction
1160     * @throws IllegalArgumentException if {@code consistency} is null
1161     * @throws IllegalStateException if this instance is not {@linkplain #start started} or in the process of shutting down
1162     */
1163    public synchronized RaftKVTransaction createTransaction(Consistency consistency) {
1164
1165        // Sanity check
1166        assert this.checkState();
1167        Preconditions.checkState(consistency != null, "null consistency");
1168        Preconditions.checkState(this.role != null, "not started");
1169        Preconditions.checkState(!this.shuttingDown, "shutting down");
1170
1171        // Base transaction on the most recent log entry (if !committed). This is itself a form of optimistic locking: we assume
1172        // that the most recent log entry has a high probability of being committed (in the Raft sense), which is of course
1173        // required in order to commit any transaction based on it.
1174        final MostRecentView view = new MostRecentView(this, consistency.isBasedOnCommittedLogEntry());
1175
1176        // Create transaction
1177        final RaftKVTransaction tx = new RaftKVTransaction(this,
1178          view.getTerm(), view.getIndex(), view.getSnapshot(), view.getView());
1179        tx.setConsistency(consistency);
1180        tx.setTimeout(this.commitTimeout);
1181        if (this.log.isDebugEnabled())
1182            this.debug("created new transaction " + tx);
1183        this.openTransactions.put(tx.getTxId(), tx);
1184
1185        // Done
1186        return tx;
1187    }

1189    /**
1190     * Commit a transaction.
1191     */
1192    void commit(final RaftKVTransaction tx) {
1193        try {
1194
1195            // Mark transaction as "commit ready" - service thread will do the rest
1196            synchronized (this) {
1197
1198                // Sanity check
1199                assert this.checkState();
1200                assert this.role != null;
1201
1202                // Check tx state
1203                switch (tx.getState()) {
1204                case EXECUTING:
1205
1206                    // Transition to COMMIT_READY state
1207                    if (this.log.isDebugEnabled())
1208                        this.debug("committing transaction " + tx);
1209                    tx.setState(TxState.COMMIT_READY);
1210                    this.requestService(new CheckReadyTransactionService(this.role, tx));
1211
1212                    // Setup commit timer
1213                    final int timeout = tx.getTimeout();
1214                    if (timeout != 0) {
1215                        final Timer commitTimer = new Timer(this, "commit timer for " + tx,
1216                          new Service("commit timeout for tx#" + tx.getTxId()) {
1217                            @Override
1218                            public void run() {
1219                                switch (tx.getState()) {
1220                                case COMMIT_READY:
1221                                case COMMIT_WAITING:
1222                                    RaftKVDatabase.this.fail(tx, new RetryTransactionException(tx,
1223                                      "transaction failed to complete within " + timeout + "ms (in state " + tx.getState() + ")"));
1224                                    break;
1225                                default:
1226                                    break;
1227                                }
1228                            }
1229                        });
1230                        commitTimer.timeoutAfter(tx.getTimeout());
1231                        tx.setCommitTimer(commitTimer);
1232                    }
1233                    break;
1234                case CLOSED:                                        // this transaction has already been committed or rolled back
1235                    throw new StaleTransactionException(tx);
1236                default:                                            // another thread is already doing the commit
1237                    this.warn("simultaneous commit()'s requested for " + tx + " by two different threads");
1238                    break;
1239                }
1240            }
1241
1242            // Wait for completion
1243            try {
1244                tx.getCommitFuture().get();
1245            } catch (InterruptedException e) {
1246                throw new RetryTransactionException(tx, "thread interrupted while waiting for commit", e);
1247            } catch (ExecutionException e) {
1248                final Throwable cause = e.getCause();
1249                ThrowableUtil.prependCurrentStackTrace(cause);
1250                if (cause instanceof Error)
1251                    throw (Error)cause;
1252                if (cause instanceof RuntimeException)
1253                    throw (RuntimeException)cause;
1254                throw new KVTransactionException(tx, "commit failed", cause);           // should never get here
1255            }
1256        } finally {
1257            this.cleanupTransaction(tx);
1258        }
1259    }

1262     * Rollback a transaction.
1263     */
1264    synchronized void rollback(RaftKVTransaction tx) {
1265
1266        // Sanity check
1267        assert this.checkState();
1268        assert this.role != null;
1269
1270        // Check tx state
1271        switch (tx.getState()) {
1272        case EXECUTING:
1273            if (this.log.isDebugEnabled())
1274                this.debug("rolling back transaction " + tx);
1275            this.cleanupTransaction(tx);
1276            break;
1277        case CLOSED:
1278            break;
1279        default:                                            // another thread is currently committing!
1280            this.warn("simultaneous commit() and rollback() requested for " + tx + " by two different threads");
1281            break;
1282        }
1283    }

1285    private synchronized void cleanupTransaction(RaftKVTransaction tx) {
1286
1287        // Debug
1288        if (this.log.isTraceEnabled())
1289            this.trace("cleaning up transaction " + tx);
1290
1291        // Do any per-role cleanups
1292        if (this.role != null)
1293            this.role.cleanupForTransaction(tx);
1294
1295        // Cancel commit timer
1296        final Timer commitTimer = tx.getCommitTimer();
1297        if (commitTimer != null)
1298            commitTimer.cancel();
1299
1300        // Remove from open transactions set
1301        this.openTransactions.remove(tx.getTxId());
1302
1303        // Transition to CLOSED
1304        tx.setState(TxState.CLOSED);
1305
1306        // Notify waiting thread if doing shutdown
1307        if (this.shuttingDown)
1308            this.notify();
1309    }

1349     *
1350     * <p>
1351     * If {@code service} has an associated {@link Role}, and that {@link Role} is no longer active

1354     * @param service the service to perform
1355     */
1356    void requestService(Service service) {

1358        assert service != null;
1359        if (!this.pendingService.add(service) || this.performingService)
1360            return;
1361        try {
1362            this.serviceExecutor.submit(new Runnable() {
1363                @Override
1364                public void run() {
1365                    RaftKVDatabase.this.handlePendingService();
1366                }
1367            });
1368        } catch (RejectedExecutionException e) {
1369            if (!this.shuttingDown)
1370                this.warn("service executor task rejected, skipping", e);

1374    // Performs pending service requests (do not invoke directly)
1375    private synchronized void handlePendingService() {
1376
1377        // Sanity check
1378        assert this.checkState();
1379        if (this.role == null)
1380            return;
1381
1382        // While there is work to do, do it
1383        this.performingService = true;
1384        try {
1385            while (!this.pendingService.isEmpty()) {
1386                final Iterator<Service> i = this.pendingService.iterator();
1387                final Service service = i.next();
1388                i.remove();
1389                assert service != null;
1390                assert service.getRole() == null || service.getRole() == this.role;
1391                if (this.log.isTraceEnabled())
1392                    this.trace("SERVICE [" + service + "] in " + this.role);
1393                try {
1394                    service.run();
1395                } catch (Throwable t) {
1396                    RaftKVDatabase.this.error("exception in " + service, t);
1397                }
1398            }
1399        } finally {
1400            this.performingService = false;
1401        }
1402    }

1421    /**
1422     * Perform a state machine flip-flop operation. Normally this would happen after a successful snapshot install.
1423     */
1424    boolean flipFlopStateMachine(long term, long index, Map<String, String> config) {
1425
1426        // Sanity check
1427        assert Thread.holdsLock(this);
1428        assert term >= 0;
1429        assert index >= 0;
1430        if (this.log.isDebugEnabled())
1431            this.debug("performing state machine flip-flop to " + index + "t" + term + " with config " + config);
1432        if (config == null)
1433            config = new HashMap<String, String>(0);
1434
1435        // Prepare updates
1436        final Writes writes = new Writes();
1437        writes.getPuts().put(LAST_APPLIED_TERM_KEY, LongEncoder.encode(term));
1438        writes.getPuts().put(LAST_APPLIED_INDEX_KEY, LongEncoder.encode(index));
1439        writes.getPuts().put(LAST_APPLIED_CONFIG_KEY, this.encodeConfig(config));
1440        writes.getPuts().put(FLIP_FLOP_KEY, this.encodeBoolean(!this.flipflop));
1441
1442        // Update persistent store
1443        try {
1444            this.kv.mutate(writes, true);
1445        } catch (Exception e) {
1446            this.error("flip-flop error updating key/value store term/index to " + index + "t" + term, e);
1447            return false;
1448        }
1449
1450        // Delete all unapplied log files (no longer applicable)
1451        this.raftLog.clear();
1452        for (File file : this.logDir.listFiles()) {
1453            if (LogEntry.LOG_FILE_PATTERN.matcher(file.getName()).matches() && !file.delete())
1454                this.error("failed to delete log file " + file);
1455        }
1456
1457        // Update in-memory copy of persistent state
1458        this.flipflop = !this.flipflop;
1459        this.lastAppliedTerm = term;
1460        this.lastAppliedIndex = index;
1461        this.lastAppliedConfig = config;
1462        this.commitIndex = this.lastAppliedIndex;
1463        this.currentConfig = this.buildCurrentConfig();
1464        this.info("new cluster configuration: " + this.currentConfig);
1465
1466        // Discard the flip-flopped state machine
1467        this.discardFlipFloppedStateMachine();
1468
1469        // Trigger key watches
1470        this.requestService(this.role.triggerKeyWatchesService);
1471
1472        // Done
1473        return true;
1474    }

1701    synchronized boolean sendMessage(Message msg) {
1702
1703        // Sanity check
1704        assert Thread.holdsLock(this);
1705
1706        // Get peer's address; if unknown, use the return address of the message being processed (if any)
1707        final String peer = msg.getRecipientId();
1708        String address = this.currentConfig.get(peer);
1709        if (address == null)
1710            address = this.returnAddress;
1711        if (address == null) {
1712            this.warn("can't send " + msg + " to unknown peer \"" + peer + "\"");
1713            return false;
1714        }
1715
1716        // Send message
1717        if (this.log.isTraceEnabled())
1718            this.trace("XMIT " + msg + " to " + address);
1719        if (this.network.send(address, msg.encode())) {
1720            this.transmitting.add(address);
1721            return true;
1722        }
1723
1724        // Transmit failed
1725        this.warn("transmit of " + msg + " to \"" + peer + "\" failed locally");
1726        return false;
1727    }

1729    synchronized void receiveMessage(String address, Message msg) {
1730
1731        // Sanity check
1732        assert Thread.holdsLock(this);
1733        assert this.checkState();
1734        if (this.role == null) {
1735            if (this.log.isDebugEnabled())
1736                this.debug("rec'd " + msg + " rec'd in shutdown state; ignoring");
1737            return;
1738        }
1739
1740        // Sanity check cluster ID
1741        if (msg.getClusterId() == 0) {
1742            this.warn("rec'd " + msg + " with zero cluster ID from " + address + "; ignoring");
1743            return;
1744        }
1745        if (this.clusterId != 0 && msg.getClusterId() != this.clusterId) {
1746            this.warn("rec'd " + msg + " with foreign cluster ID "
1747              + String.format("0x%08x", msg.getClusterId()) + " != " + String.format("0x%08x", this.clusterId) + "; ignoring");
1748            return;
1749        }
1750
1751        // Sanity check sender
1752        final String peer = msg.getSenderId();
1753        if (peer.equals(this.identity)) {
1754            this.warn("rec'd " + msg + " from myself (\"" + peer + "\", address " + address + "); ignoring");
1755            return;
1756        }
1757
1758        // Sanity check recipient
1759        final String dest = msg.getRecipientId();
1760        if (!dest.equals(this.identity)) {
1761            this.warn("rec'd misdirected " + msg + " intended for \"" + dest + "\" from " + address + "; ignoring");
1762            return;
1763        }
1764
1765        // Is my term too low? If so update and revert to follower
1766        if (msg.getTerm() > this.currentTerm) {
1767
1768            // First check with current role; in some special cases we ignore this
1769            if (!this.role.mayAdvanceCurrentTerm(msg)) {
1770                if (this.log.isTraceEnabled()) {
1771                    this.trace("rec'd " + msg + " with term " + msg.getTerm() + " > " + this.currentTerm + " from \""
1772                      + peer + "\" but current role says to ignore it");
1773                }
1774                return;
1775            }
1776
1777            // Revert to follower
1778            if (this.log.isDebugEnabled()) {
1779                this.debug("rec'd " + msg.getClass().getSimpleName() + " with term " + msg.getTerm() + " > "
1780                  + this.currentTerm + " from \"" + peer + "\", updating term and "
1781                  + (this.role instanceof FollowerRole ? "remaining a" : "reverting to") + " follower");
1782            }
1783            if (!this.advanceTerm(msg.getTerm()))
1784                return;
1785            this.changeRole(msg.isLeaderMessage() ? new FollowerRole(this, peer, address) : new FollowerRole(this));
1786        }
1787
1788        // Is sender's term too low? Ignore it (except ping request)
1789        if (msg.getTerm() < this.currentTerm && !(msg instanceof PingRequest)) {
1790            if (this.log.isDebugEnabled()) {
1791                this.debug("rec'd " + msg + " with term " + msg.getTerm() + " < " + this.currentTerm
1792                  + " from \"" + peer + "\" at " + address + ", ignoring");
1793            }
1794            return;
1795        }
1796
1797        // Debug
1798        if (this.log.isTraceEnabled())
1799            this.trace("RECV " + msg + " in " + this.role + " from " + address);
1800
1801        // Handle message
1802        this.returnAddress = address;
1803        try {
1804            msg.visit(new MessageSwitch() {
1805                @Override
1806                public void caseAppendRequest(AppendRequest msg) {
1807                    RaftKVDatabase.this.role.caseAppendRequest(msg);
1808                }
1809                @Override
1810                public void caseAppendResponse(AppendResponse msg) {
1811                    RaftKVDatabase.this.role.caseAppendResponse(msg);
1812                }
1813                @Override
1814                public void caseCommitRequest(CommitRequest msg) {
1815                    RaftKVDatabase.this.role.caseCommitRequest(msg);
1816                }
1817                @Override
1818                public void caseCommitResponse(CommitResponse msg) {
1819                    RaftKVDatabase.this.role.caseCommitResponse(msg);
1820                }
1821                @Override
1822                public void caseGrantVote(GrantVote msg) {
1823                    RaftKVDatabase.this.role.caseGrantVote(msg);
1824                }
1825                @Override
1826                public void caseInstallSnapshot(InstallSnapshot msg) {
1827                    RaftKVDatabase.this.role.caseInstallSnapshot(msg);
1828                }
1829                @Override
1830                public void casePingRequest(PingRequest msg) {
1831                    RaftKVDatabase.this.role.casePingRequest(msg);
1832                }
1833                @Override
1834                public void casePingResponse(PingResponse msg) {
1835                    RaftKVDatabase.this.role.casePingResponse(msg);
1836                }
1837                @Override
1838                public void caseRequestVote(RequestVote msg) {
1839                    RaftKVDatabase.this.role.caseRequestVote(msg);
1840                }
1841            });
1842        } finally {
1843            this.returnAddress = null;
1844        }
1845    }

1895    Map<String, String> decodeConfig(byte[] key) throws IOException {
1896        final Map<String, String> config = new HashMap<>();
1897        final byte[] value = this.kv.get(key);
1898        if (value == null)

1900        try {

1902            while (true) {

1905                    break;
1906                data.reset();
1907                config.put(data.readUTF(), data.readUTF());
1908            }
1909        } catch (IOException e) {
1910            throw new IOException("can't interpret encoded config "
1911              + ByteUtil.toString(value) + " under key " + ByteUtil.toString(key), e);
1912        }
1913        return config;
1914    }

1916    byte[] encodeConfig(Map<String, String> config) {
1917        final ByteArrayOutputStream buf = new ByteArrayOutputStream();
1918        final DataOutputStream data = new DataOutputStream(buf);
1919        try {
1920            for (Map.Entry<String, String> entry : config.entrySet()) {
1921                data.writeUTF(entry.getKey());
1922                data.writeUTF(entry.getValue());
1923            }
1924            data.flush();
1925        } catch (IOException e) {
1926            throw new RuntimeException("unexpected error", e);
1927        }
1928        return buf.toByteArray();
1929    }

1975    private boolean checkState() {
1976        try {
1977            this.doCheckState();
1978        } catch (AssertionError e) {
1979            throw new AssertionError("checkState() failure for " + this, e);
1980        }
1981        return true;
1982    }
1983
1984    private void doCheckState() {
1985        assert Thread.holdsLock(this);
1986
1987        // Handle stopped state
1988        if (this.role == null) {
1989            assert this.random == null;
1990            assert this.currentTerm == 0;
1991            assert this.commitIndex == 0;
1992            assert this.lastAppliedTerm == 0;
1993            assert this.lastAppliedIndex == 0;
1994            assert this.lastAppliedConfig == null;
1995            assert this.currentConfig == null;
1996            assert this.clusterId == 0;
1997            assert this.raftLog.isEmpty();
1998            assert this.logDirChannel == null;
1999            assert this.serviceExecutor == null;
2000            assert this.keyWatchTracker.getNumKeysWatched() == 0;
2001            assert this.transmitting.isEmpty();
2002            assert this.openTransactions.isEmpty();
2003            assert this.pendingService.isEmpty();
2004            assert !this.shuttingDown;
2005            return;
2006        }
2007
2008        // Handle running state
2009        assert this.kv != null;
2010        assert this.random != null;
2011        assert this.serviceExecutor != null;
2012        assert this.logDirChannel != null;
2013        assert !this.serviceExecutor.isShutdown() || this.shuttingDown;
2014
2015        assert this.currentTerm >= 0;
2016        assert this.commitIndex >= 0;
2017        assert this.lastAppliedTerm >= 0;
2018        assert this.lastAppliedIndex >= 0;
2019        assert this.lastAppliedConfig != null;
2020        assert this.currentConfig != null;
2021
2022        assert this.currentTerm >= this.lastAppliedTerm;
2023        assert this.commitIndex >= this.lastAppliedIndex;
2024        assert this.commitIndex <= this.lastAppliedIndex + this.raftLog.size();
2025        assert this.keyWatchIndex <= this.commitIndex;
2026        long index = this.lastAppliedIndex;
2027        long term = this.lastAppliedTerm;
2028        for (LogEntry logEntry : this.raftLog) {
2029            assert logEntry.getIndex() == index + 1;
2030            assert logEntry.getTerm() >= term;
2031            index = logEntry.getIndex();
2032            term = logEntry.getTerm();
2033        }
2034
2035        // Check configured vs. unconfigured
2036        if (this.isConfigured()) {
2037            assert this.clusterId != 0;
2038            assert this.currentTerm > 0;
2039            assert this.lastAppliedTerm >= 0;
2040            assert this.lastAppliedIndex >= 0;
2041            assert !this.currentConfig.isEmpty();
2042            assert this.currentConfig.equals(this.buildCurrentConfig());
2043            assert this.getLastLogTerm() > 0;
2044            assert this.getLastLogIndex() > 0;
2045        } else {
2046            assert this.lastAppliedTerm == 0;
2047            assert this.lastAppliedIndex == 0;
2048            assert this.lastAppliedConfig.isEmpty();
2049            assert this.currentConfig.isEmpty();
2050            assert this.raftLog.isEmpty();
2051        }
2052
2053        // Check role
2054        assert this.role.checkState();
2055
2056        // Check transactions
2057        for (RaftKVTransaction tx : this.openTransactions.values())
2058            tx.checkStateOpen(this.currentTerm, this.getLastLogIndex(), this.commitIndex);
2059    }
